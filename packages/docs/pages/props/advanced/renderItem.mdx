# Render Item

The `renderItem` prop allows for complete customization of how each chip is rendered in the Chipster component.

## Definition

| Property | Description |
| -------- | ----------- |
| Type     | `(item: ChipsterItem, index: number, isHighlighted: boolean) => React.ReactNode` |
| Default  | `undefined` |

## Usage

Here's an advanced example where we create a custom chip for user information:

```jsx filename="AdvancedChipsterExample.jsx" {3-41} copy
import React from 'react';
import { Chipster } from '@micoblanc/chipster';
import { FaUser, FaEnvelope, FaPhone, FaTimes } from 'react-icons/fa';

function AdvancedChipsterExample() {
  const users = [
    { id: '1', name: 'John Doe', email: 'john@example.com', phone: '123-456-7890' },
    { id: '2', name: 'Jane Smith', email: 'jane@example.com', phone: '098-765-4321' },
  ];

  const renderItem = (item, index, isHighlighted) => {
    const user = JSON.parse(item.text);
    return (
      <div className={`user-chip ${isHighlighted ? 'highlighted' : ''}`}>
        <div className="user-icon">
          <FaUser />
        </div>
        <div className="user-info">
          <div className="user-name">{user.name}</div>
          <div className="user-details">
            <span className="user-email">
              <FaEnvelope />
              {user.email}
            </span>
            <span className="user-phone">
              <FaPhone />
              {user.phone}
            </span>
          </div>
        </div>
        <button 
          onClick={() => console.log(`Remove ${user.name}`)}
          className="remove-button"
          aria-label={`Remove ${user.name}`}
        >
          <FaTimes />
        </button>
      </div>
    );
  };

  return (
    <Chipster
      placeholder="Add a user..."
      renderItem={renderItem}
      transform={(value) => JSON.stringify(users.find(user => user.name.toLowerCase().includes(value.toLowerCase())))}
    />
  );
}
````


## Notes

- This example demonstrates how to use `renderItem` to create a complex chip structure for user information.
- The chip displays the user's name, email, and phone number, along with appropriate icons.
- We've used class names instead of inline styles. You would need to define these classes in your CSS file.
- The `transform` prop is used to convert the input value into a JSON string of the full user object.
- When rendering, we parse the JSON string back into an object to access all user properties.

## Advanced Usage Tips

1. **Data Transformation**: The `transform` prop is used here to store complex data structures as strings.
2. **Conditional Rendering**: The `isHighlighted` parameter allows for different styling or structure based on the chip's state.
3. **Accessibility**: Note the use of `aria-label` for the remove button to enhance accessibility.
4. **Custom Interactions**: The remove button demonstrates how to add custom interactions within the chip.
